# VoC Monthly Report Automation

의료 AI 서비스의 고객의 소리(Voice of Customer, VoC)를 월간 리포트로 자동화하기 위해 설계한 Claude Skill 기반 업무 자동화 프로젝트입니다.

이 저장소는 실제 회사 데이터가 아닌 mock-up 데이터로 재구성한 포트폴리오입니다. 병원명, 문의 내용, 분류 체계, 리포트 수치는 모두 공개 가능한 형태로 변형하거나 새로 작성했습니다.

## 프로젝트 개요

의료 AI 현장에서 간호사로 근무하며 고객 대응, 제품 피드백 정리, 유관 부서 커뮤니케이션을 함께 수행하다 보면 VoC는 단순한 문의 기록이 아니라 제품 개선의 중요한 근거가 됩니다. 하지만 VoC 데이터는 표현이 다양하고 맥락 의존적이며, 같은 요구가 병원마다 다른 언어로 접수되는 경우가 많습니다.

기존 월간 보고 과정은 엑셀 정리, 유사 의견 병합, 카테고리 분류, 주요 이슈 선별, 보고서 작성이 대부분 수작업으로 이루어졌습니다. 이 과정은 반복 시간이 길고, 담당자의 해석 기준에 따라 분류 결과가 달라질 수 있으며, 전월 대비 추적이나 미해결 이슈 관리에도 한계가 있었습니다.

이 프로젝트는 이러한 운영 문제를 해결하기 위해 VoC 원장을 구조화하고, Claude를 활용해 의미 기반 분류와 요약을 수행하며, 월간 리포트를 자동 생성할 수 있는 흐름을 설계한 사례입니다.

## 왜 자동화가 필요했는가

VoC 리포트 자동화의 목적은 단순히 보고서 작성 시간을 줄이는 데 있지 않습니다. 더 중요한 목적은 고객 피드백이 제품, 임상 운영, CS, 개발 조직 사이에서 일관된 기준으로 해석되고 누적되도록 만드는 것입니다.

반복 작업을 자동화함으로써 담당자는 데이터 복사와 문서 편집보다 이슈의 우선순위 판단, 임상적 맥락 해석, 제품 개선 가능성 검토에 더 많은 시간을 사용할 수 있습니다.

또한 매월 동일한 기준으로 VoC를 분류하고 집계하면 다음과 같은 운영상의 장점이 생깁니다.

- 유사한 VoC를 의미 단위로 병합하여 중복 이슈를 줄입니다.
- 미해결, 반영 예정, 완료 상태를 월별로 추적할 수 있습니다.
- 고객 문의가 단순 CS 이슈인지 제품 개선 요구인지 구분할 수 있습니다.
- 경영진, 개발팀, 운영팀이 같은 기준의 리포트를 공유할 수 있습니다.
- 누적된 VoC를 제품 로드맵과 고객 교육 자료 개선에 연결할 수 있습니다.

## 자동화 흐름

```text
Notion VoC DB
  -> CSV export
  -> Excel master workbook
  -> Claude Skill processing
  -> Monthly HTML/PDF report
  -> Review, feedback, taxonomy update
```

현재 mock-up은 다음 산출물로 구성되어 있습니다.

| 파일 | 설명 |
| --- | --- |
| `VoC_master_mock_up.xlsx` | VoC 원장, raw 데이터, 긍정 VoC, 단순 문의, 월별 집계, 종합 요약을 포함한 마스터 워크북 |
| `VoC_Report_2026-06_mock_up.html` | 월간 VoC 리포트 mock-up. 브라우저에서 확인하고 PDF로 출력할 수 있는 HTML 보고서 |
| `README.md` | 프로젝트 배경, 설계 의도, 활용 계획을 정리한 포트폴리오 문서 |

## 리포트 구성

월간 리포트는 현업 의사결정에 바로 사용할 수 있도록 다음 섹션으로 구성했습니다.

1. 6월 VoC 종합 요약
2. 미진행 VoC 현황
3. 대분류 키워드별 VoC 구성
4. 반영 예정 VoC 리스트
5. 담당자 Pick 및 주목해야 할 VoC
6. 자주 묻는 문의 TOP 5
7. 긍정 VoC

각 섹션은 단순 현황 나열보다 "무엇을 먼저 봐야 하는지", "어떤 부서와 논의해야 하는지", "다음 달에 어떤 상태 변화를 추적해야 하는지"가 드러나도록 설계했습니다.

## Claude Skill 설계 포인트

### 1. 의미 기반 VoC 그룹핑

VoC는 동일한 불편이나 요구라도 표현 방식이 다릅니다. 예를 들어 "점수 산출 근거를 알고 싶다", "AI 점수가 왜 바뀌었는지 설명이 필요하다"는 문장은 단어는 다르지만 같은 제품 개선 요구로 볼 수 있습니다.

Claude Skill은 단순 키워드 매칭이 아니라 문장의 의미와 업무 맥락을 기준으로 유사 VoC를 묶도록 설계했습니다. 이를 통해 중복 의견을 줄이고, 한 병원의 단발성 문의가 아니라 여러 고객에게 반복되는 구조적 이슈를 발견할 수 있습니다.

### 2. 대표 문장 생성

유사 VoC를 묶은 뒤에는 특정 고객의 표현 하나를 그대로 대표 문장으로 사용하는 대신, 그룹 내 의견을 종합해 보고서용 문장으로 재작성합니다. 이 방식은 개별 의견의 뉘앙스를 보존하면서도 경영진과 제품팀이 빠르게 이해할 수 있는 형태로 정리하는 데 목적이 있습니다.

### 3. 분류 기준의 누적 관리

AI 분류 결과는 매월 새로 판단하면 기준이 흔들릴 수 있습니다. 따라서 카테고리 정의, 그룹 정의, 상태 기준을 별도로 관리하고, 신규 VoC가 기존 그룹에 포함되는지 또는 새 그룹으로 만들어야 하는지를 검토하는 구조가 필요합니다.

이 프로젝트는 human-in-the-loop 방식을 전제로 합니다. Claude가 초안을 제안하고, 담당자가 애매한 항목을 검토하며, 확정된 기준은 다음 달 처리 규칙에 반영합니다.

### 4. 월별 스냅샷과 상태 추적

VoC 리포트는 특정 월의 현황표로 끝나면 가치가 제한됩니다. 미해결 이슈가 다음 달에 해결되었는지, 반영 예정 항목이 실제 제품 개선으로 이어졌는지, 반복 문의가 고객 교육으로 감소했는지 추적할 수 있어야 합니다.

이를 위해 월별 스냅샷을 남기고, 상태 변화와 신규/반복 이슈를 비교할 수 있는 구조로 확장할 예정입니다.

## 향후 활용 계획

이 자동화 흐름은 월간 보고서 생성 도구를 넘어, 의료 AI 서비스 운영 품질을 개선하는 피드백 시스템으로 발전시키는 것을 목표로 합니다.

향후에는 다음 방향으로 확장할 계획입니다.

- Notion DB 또는 CSV export와 연동해 월간 리포트 초안을 자동 생성
- 반복 문의를 FAQ, 고객 교육 자료, 온보딩 자료 개선에 연결
- 제품 개선 요청을 기능 단위 backlog와 연결
- 미해결 VoC의 aging, 심각도, 영향 병원 수를 추적
- 월별 카테고리 변화와 전월 대비 증가 이슈를 자동 감지
- 담당자 검토 결과를 다시 분류 기준과 프롬프트에 반영

## 피드백 수렴 및 변경 기록

이 프로젝트는 한 번 만든 자동화 도구가 아니라, 실제 운영 피드백을 반영해 계속 개선하는 업무 시스템으로 관리할 예정입니다.

매월 리포트 생성 후 다음 항목을 기록합니다.

- 잘못 분류된 VoC와 수정 사유
- 새로 추가된 카테고리 또는 병합된 카테고리
- 담당자가 수동으로 보정한 요약 문장
- 개발팀, CS팀, 경영진이 요청한 리포트 개선 사항
- 다음 달 Skill 프롬프트 또는 처리 규칙에 반영할 변경점

이 기록은 추후 `CHANGELOG.md` 또는 `docs/feedback-log.md`로 분리해 관리할 수 있습니다. 목적은 자동화 결과를 그대로 신뢰하는 것이 아니라, AI가 만든 초안을 현업 기준으로 검증하고 개선하는 운영 루프를 남기는 것입니다.

## 포트폴리오 공개 방식

GitHub에는 다음 구조로 공개하는 것을 권장합니다.

```text
voc-report-automation/
  README.md
  samples/
    VoC_master_mock_up.xlsx
    VoC_Report_2026-06_mock_up.html
  docs/
    workflow.md
    taxonomy.md
    feedback-log.md
  skill/
    prompt.md
    instructions.md
  CHANGELOG.md
```

포트폴리오에서는 코드의 양보다 문제 정의와 운영 설계가 더 중요합니다. 따라서 README에는 전체 스토리와 결과물을 보여주고, `docs/`에는 분류 기준과 변경 이력을, `skill/`에는 공개 가능한 범위의 프롬프트 설계 원칙을 정리하는 방식이 적합합니다.

## 사용 도구

- Claude: VoC 분류, 의미 기반 그룹핑, 요약, 보고서 문장 생성
- Excel: VoC 원장 관리, 월별 집계, 검토용 데이터 테이블
- HTML/CSS: 월간 보고서 시각화 및 PDF 출력
- Notion DB: 원본 VoC 수집 및 운영 데이터 관리

## 데이터 및 보안

이 프로젝트는 포트폴리오 공개를 위해 실제 데이터와 분리된 mock-up 데이터로 구성했습니다. 실제 운영 환경에 적용할 경우 고객사명, 사용자 정보, 병원별 세부 문의, 제품 내부 정보는 비식별화하거나 접근 권한이 제한된 환경에서만 처리해야 합니다.

의료 AI 서비스의 VoC는 제품 피드백인 동시에 고객 운영 정보가 될 수 있으므로, 자동화 과정에서도 데이터 최소화, 비식별화, 접근 권한 관리, 결과물 검토 절차를 함께 고려해야 합니다.
